# Product Principles

These principles govern how FortuneSats should evolve. They exist to protect the product's identity as the system grows more capable.

## Core Identity

FortuneSats is a fortune oracle. You pay sats, you receive wisdom. The interaction is a ritual, not a transaction.

## Human-Centric Constraints

1. **Simplicity over features.** Every screen should do one thing well. If a feature requires explanation, it's too complex.

2. **Ritual over utility.** The pay → wait → reveal → react sequence is sacred. It should feel like opening a fortune cookie, not calling an API.

3. **Delight over decoration.** Animations, rarity reveals, confetti, and the 3D dragon exist because they make the moment feel special. They are not filler.

4. **Premium over busy.** The app should feel premium and minimal. No dashboards, no admin panels, no settings pages, no onboarding flows.

5. **Quiet confidence.** FortuneSats doesn't need to explain itself. The product is the experience.

## Agent-Ready, Not Agent-Visible

The agent infrastructure should stay under the hood:

- **No agent dashboards** in the UI
- **No API documentation links** on the homepage
- **No developer-facing controls** visible to casual users
- **No toggle switches** or config panels in the product

The agent layer exists for machines. Humans should never encounter it unless they're a developer reading the docs.

## Payment Philosophy

- **Value for value.** 100 sats is a real payment. It's not a demo, not a freemium hook, not a microtransaction trick.
- **Lightning first.** Instant, global, permissionless. The payment should feel as fast as the fortune reveal.
- **L402 for machines.** When agents pay, they pay the same way: real sats for real value. No free tiers for bots.
- **Fair pricing.** The price should feel fair and fun, never extractive.

## Growth Guardrails

When adding new features, ask:

1. Does this make the fortune moment better?
2. Does this make the app feel busier or heavier?
3. Would a first-time user understand this without explanation?
4. Does this serve the product, or does it serve the developer?

If a feature fails any of these, reconsider.

## What FortuneSats Is Not

- Not a wisdom aggregator or quote database
- Not a developer tool or API product
- Not a gamification platform (leaderboard and collections serve the ritual, not the reverse)
- Not a crypto project — it's a product that uses Bitcoin as a payment rail
