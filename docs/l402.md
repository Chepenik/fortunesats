# L402 Payment Protocol

L402 (formerly LSAT) is a standard for HTTP-native machine payments over the Lightning Network. FortuneSats uses L402 to gate agent-facing endpoints, enabling machines to pay for access the same way humans pay through the UI.

## How L402 Works

L402 extends HTTP 402 (Payment Required) with a Lightning invoice flow:

```
Agent                              FortuneSats
  │                                     │
  │  GET /api/agent/fortune             │
  │────────────────────────────────────>│
  │                                     │
  │  402 Payment Required               │
  │  WWW-Authenticate: L402             │
  │    macaroon="<macaroon>"            │
  │    invoice="<bolt11_invoice>"       │
  │<────────────────────────────────────│
  │                                     │
  │  [Agent pays Lightning invoice]     │
  │  [Receives preimage as proof]       │
  │                                     │
  │  GET /api/agent/fortune             │
  │  Authorization: L402                │
  │    <macaroon>:<preimage>            │
  │────────────────────────────────────>│
  │                                     │
  │  200 OK                             │
  │  { fortune: { ... } }              │
  │<────────────────────────────────────│
```

## Key Concepts

### Macaroon
A bearer credential issued by the server. Contains caveats (conditions) such as expiry time, endpoint scope, or usage limits. The macaroon alone is not sufficient — it must be paired with a payment proof.

### Invoice
A standard BOLT-11 Lightning invoice. The agent's Lightning wallet pays this invoice and receives a preimage (proof of payment).

### Preimage
The cryptographic proof that the invoice was paid. Combined with the macaroon, it forms a complete L402 credential.

## FortuneSats L402 Design

### Current State

L402 gating is **designed but not yet active** (`FS_L402=false` by default). The infrastructure is in place:

- `src/lib/l402.ts` — Payment gate abstraction with `withAgentPayment()` wrapper
- `src/lib/config.ts` — Feature flag to enable/disable L402
- `src/app/api/agent/fortune/route.ts` — Endpoint wired through the L402 gate

When L402 is disabled, the agent endpoint serves fortunes freely (useful for development, testing, and initial adoption).

### Enabling L402

Set the environment variable:

```bash
FS_L402=true
```

When enabled, the `withAgentPayment()` wrapper activates and:

1. Checks for an `Authorization: L402` header on incoming requests
2. If absent, returns 402 with payment instructions
3. If present, validates the credential and passes the request through

### Pricing

| Access Type        | Price    | Payment Method  |
|-------------------|----------|-----------------|
| Single fortune (human) | 100 sats | Lightning (MDK) |
| Single fortune (agent) | 100 sats | L402            |
| Fortune pack (human)   | ~10,000 sats | On-chain Bitcoin |

Agent pricing matches human pricing. The value exchange is the same regardless of who is consuming it.

### Future: Full MDK Integration

The `withAgentPayment()` abstraction is designed to delegate to MoneyDevKit's `withPayment` when L402 is enabled. The integration path:

```typescript
// Current: l402.ts handles the 402 response format
// Future: Delegate to MDK for full invoice generation + validation
import { withPayment } from "@moneydevkit/nextjs/server";

// withAgentPayment will call:
withPayment({ amount: 100, currency: "SAT" }, handler);
```

MDK already handles L402 for the human-facing `/api/fortune` endpoint. The same infrastructure extends to agent endpoints.

## Agent Implementation Guide

### For Agent Developers

If you're building an agent that consumes FortuneSats:

1. **Check if L402 is active**: The `pricing.l402_enabled` field in the response tells you.
2. **Handle 402 gracefully**: Parse the `WWW-Authenticate` header for invoice details.
3. **Use a Lightning wallet**: Your agent needs access to a Lightning node or wallet API to pay invoices.
4. **Cache the credential**: L402 macaroons may be reusable within their caveat constraints.
5. **Retry with auth**: After payment, retry the same request with the `Authorization: L402` header.

```python
if response.status_code == 402:

    auth_header = response.headers["WWW-Authenticate"]
    # Extract macaroon and invoice from header

    # Step 3: Pay invoice via your Lightning wallet
    preimage = your_lightning_wallet.pay(invoice)

    # Step 4: Retry with L402 credential
    response = requests.get(
        "https://fortunesats.vercel.app/api/agent/fortune",
        headers={"Authorization": f"L402 {macaroon}:{preimage}"}
    )

fortune = response.json()["fortune"]
```

### Example: curl

```bash
# Without L402 (when disabled)
curl https://fortunesats.vercel.app/api/agent/fortune

# With L402 (when enabled)
curl -H "Authorization: L402 <macaroon>:<preimage>" \
  https://fortunesats.vercel.app/api/agent/fortune
```

## Why L402 for Agents

Traditional API monetization uses API keys, subscription tiers, and billing accounts. L402 is different:

- **No accounts needed.** An agent can pay and access in a single flow.
- **No subscription overhead.** Pay per request, not per month.
- **Permissionless.** Any agent with a Lightning wallet can interact.
- **Instant settlement.** No invoicing, no net-30, no payment processors.
- **Native to Bitcoin.** Aligns with FortuneSats' identity as a Bitcoin-native product.
