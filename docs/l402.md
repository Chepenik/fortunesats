# L402 Payment Protocol

L402 (formerly LSAT) is how machines pay for things on the internet. No accounts, no API keys, no subscriptions. Just Lightning sats.

FortuneSats uses L402 to let agents pay for fortunes the same way humans do -- except instead of clicking a button, they settle a Lightning invoice.

---

## How It Works (The Short Version)

```
Your Agent                            FortuneSats
    |                                      |
    |  GET /api/agent/fortune              |
    |------------------------------------->|
    |                                      |
    |  402 Payment Required                |
    |  "Here's a Lightning invoice"        |
    |<-------------------------------------|
    |                                      |
    |  [Pays 100 sats via Lightning]       |
    |  [Gets a preimage as receipt]        |
    |                                      |
    |  GET /api/agent/fortune              |
    |  Authorization: L402 proof           |
    |------------------------------------->|
    |                                      |
    |  200 OK                              |
    |  { fortune: "The obstacle..." }      |
    |<-------------------------------------|
```

That's it. Request, pay, prove, receive. No signup flow, no OAuth dance, no billing page.

---

## Key Concepts

### Macaroon
A bearer credential from the server. Think of it as a ticket stub -- it says what you're allowed to access, but it's not valid on its own. You need to prove you paid.

### Invoice
A standard BOLT-11 Lightning invoice. Your agent's wallet pays this and gets a preimage (a cryptographic receipt) in return.

### Preimage
The proof of payment. Combine it with the macaroon, and you have a complete L402 credential. The server can verify you paid without talking to any payment processor.

## Current Status

L402 is **designed and plumbed but not yet active** (`FS_L402=false` by default).

The infrastructure is ready:
- `src/lib/l402.ts` -- Payment gate abstraction with `withAgentPayment()` wrapper
- `src/lib/config.ts` -- Feature flag to flip the switch
- `src/app/api/agent/fortune/route.ts` -- Already wired through the L402 gate

When L402 is off, the agent API serves fortunes freely -- great for development, testing, and building integrations before payment goes live.

### Turning It On

```bash
FS_L402=true
```

That's it. The `withAgentPayment()` wrapper activates and starts gating requests.

Everyone pays the same. Humans and machines get equal treatment.

| Human | Single fortune | 100 sats | Lightning via MDK |
| Human | 100-fortune pack | ~10,000 sats | On-chain Bitcoin |
| Agent | Single fortune | 100 sats | L402 |

## Building an L402-Compatible Agent

If you're building something that talks to FortuneSats, here's what you need to know:

### Step by Step

1. **Check `pricing.l402_enabled`** in any fortune response -- it tells you if payments are required
2. **Handle 402 responses** -- parse the `WWW-Authenticate` header for the macaroon and invoice
3. **Have a Lightning wallet** -- your agent needs access to a wallet or node to pay invoices
4. **Cache credentials** -- L402 macaroons may be reusable within their caveat constraints
5. **Retry with auth** -- after payment, send the same request with `Authorization: L402 <macaroon>:<preimage>`

### Python Example

```python
import requests

# Step 1: Ask for a fortune
response = requests.get("https://fortunesats.com/api/agent/fortune")

if response.status_code == 402:
    # Step 2: Parse the payment requirement
    auth_header = response.headers["WWW-Authenticate"]
    # Extract macaroon and invoice from header

    # Step 3: Pay the Lightning invoice
    preimage = your_lightning_wallet.pay(invoice)

    # Step 4: Retry with proof of payment
    response = requests.get(
        "https://fortunesats.com/api/agent/fortune",
        headers={"Authorization": f"L402 {macaroon}:{preimage}"}
    )

fortune = response.json()["fortune"]
print(f"You pulled: {fortune['text']} ({fortune['rarity']})")
```

### curl Example

```bash
# When L402 is off (free access)
curl https://fortunesats.com/api/agent/fortune

# When L402 is on (with payment proof)
curl -H "Authorization: L402 <macaroon>:<preimage>" \
  https://fortunesats.com/api/agent/fortune
```

## Why L402 Instead of API Keys?

| Traditional API | L402 |
|----------------|------|
| Create an account | Just pay |
| Get an API key | Just pay |
| Choose a subscription tier | Just pay |
| Enter billing info | Just pay |
| Wait for invoice to clear | Just pay |

L402 strips away the entire account-and-billing industrial complex. An agent with a Lightning wallet can go from zero to pulling fortunes in a single HTTP round-trip.

Other advantages:
- **Permissionless** -- any Lightning wallet works, no approval needed
- **Instant settlement** -- paid and confirmed in seconds
- **Per-request pricing** -- pay for what you use, nothing more
- **Native to Bitcoin** -- aligns with FortuneSats' identity

## Future: Full MDK Integration

The `withAgentPayment()` abstraction is designed to delegate to MoneyDevKit's `withPayment` when L402 goes live:

```typescript
// Current: l402.ts handles the 402 response format
// Future: MDK handles invoice generation + validation
import { withPayment } from "@moneydevkit/nextjs/server";

withPayment({ amount: 100, currency: "SAT" }, handler);
```

MDK already handles Lightning payments for the human-facing UI. The same infrastructure extends to agent endpoints with a single flag flip.
